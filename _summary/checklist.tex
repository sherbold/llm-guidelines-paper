\section{Reporting Checklist}
\label{sec:checklist}

% NOTE: Each \item below carries a non-rendered comment citing the source
% sentence(s) in the corresponding _guidelines/0X_*.tex file. Update both
% the item and the comment when guideline text changes.
% Ordering rule (per CLAUDE.md): within each section, sort by severity
% (\iconM before \iconS), then by reporting location
% (\paper -> unspecified -> \supplementarymaterial).

The following checklist, inspired by CONSORT~\citep{Schulz2010}, summarizes actionable items from the guidelines based on the \tldr sections.
The checklist is organized along typical paper sections.
Items marked \iconM are requirements (\must); items marked \iconS are recommendations (\should).
Each item references its source guideline (G1--G8).
Items annotated with \paper or \supplementarymaterial indicate where information should be reported; unmarked items may be reported in either.

\paragraph{\textbf{Introduction}}
\begin{itemize}[label=$\square$]
    \item \iconM Disclose any use of LLMs in the empirical study, specifying which LLM, how, and where it was used~(G1).
    % G1 Recommendations: "researchers \must clearly declare that an LLM was used"; specifies which/how/where in the research process.
    \item \iconS Report the purpose of using LLMs, automated tasks, and expected benefits in the \paper~(G1).
    % G1 Recommendations: "researchers \should report the exact purpose of using an LLM in a study, the tasks it was used to automate, and the expected benefits in the \paper".
\end{itemize}

\paragraph{\textbf{Research Design and Methods}}\mbox{}\\
\noindent\textit{Model Selection and Configuration}
\begin{itemize}[label=$\square$]
    \item \iconM Report the exact LLM model or tool version, configuration, and experiment date in the \paper~(G2).
    % G2 Recommendations: "Researchers \must document in the \paper which model or tool version they used... along with the date when the experiments were carried out and the configured parameters that affect output generation".
    \item \iconM For fine-tuned models, describe the fine-tuning goal, dataset, and procedure in the \paper~(G2).
    % G2 Recommendations: "If a model was fine-tuned, researchers \must describe the fine-tuning goal..., the fine-tuning procedure..., and the fine-tuning dataset... in the \paper".
    \item \iconS Report default parameters and explain model and version choices~(G2).
    % G2 Recommendations: "researchers \should always report all configuration values, even if they used the defaults"; "Researchers \should motivate in the \paper why they selected certain models, versions, and configurations".
    \item \iconS For quantized models, report the quantization level (e.g., 4-bit, 8-bit) and method (e.g., GPTQ or AWQ)~(G2).
    % G2 Recommendations: "When using quantized models, researchers \should report the quantization level... and method".
    \item \iconS Compare base and fine-tuned models using suitable metrics and benchmarks; share fine-tuning data and weights as \supplementarymaterial (or justify in the \paper why they cannot be shared)~(G2).
    % G2 Recommendations: "Suitable benchmarks and metrics \should be used to compare the base model with the fine-tuned model"; "Researchers \should either share the fine-tuning dataset as part of the \supplementarymaterial or explain in the \paper why the data cannot be shared. The same applies to the fine-tuned model weights".
\end{itemize}

\noindent\textit{System and Prompt Design}
\begin{itemize}[label=$\square$]
    \item \iconM Describe the full architecture of LLM-based tools in the \paper, including the role of the LLM, interactions with other components, and overall system behavior~(G3).
    % G3 Recommendations: "Researchers \must clearly describe the tool architecture and what exactly the LLM... contributes"; for complex systems "the \paper \must contain a high-level description of the system architecture".
    \item \iconM Describe any context-file mechanisms used (e.g., \texttt{AGENTS.md}, \texttt{CLAUDE.md}) in the \paper~(G3).
    % G3 Context Files: "Researchers \must describe which context-file mechanisms were used in their setup".
    \item \iconM Summarize which tools and skills were exposed to the model in the \paper~(G3).
    % G3 Tool Catalog: "Researchers \must summarize in the \paper which tools and skills were exposed to the model".
    \item \iconM Specify whether zero-shot, one-shot, or few-shot prompting was used in the \paper~(G3).
    % G3 Prompting Strategy: "Researchers \must specify whether zero-shot, one-shot, or few-shot prompting was used".
    \item \iconM If autonomous agents are used, specify agent roles, reasoning frameworks, and communication flows~(G3).
    % G3 Agent-Based Systems: "researchers \must describe its exact architecture, including the agents' roles..., whether it is a single-agent or multi-agent system, how it interacts with external tools and users, and the reasoning framework used".
    \item \iconM Report hosting, hardware setup, and latency implications~(G3).
    % G3 Recommendations: "Researchers \must clarify whether local infrastructure or cloud services were used, including detailed infrastructure specifications and latency considerations".
    \item \iconM Specify prompt reuse across models and configurations~(G3).
    % G3 Prompt Development: "researchers \must make clear which prompts were used for which models in which versions and with which configuration".
    \item \iconM For tools using retrieval or augmentation methods, describe how data is retrieved, stored, and integrated~(G3).
    % G3 Retrieval/Augmentation: "researchers \must describe how external data is retrieved, stored, and integrated into the LLM's responses".
    \item \iconM For dynamically generated prompts, document the generation process thoroughly~(G3).
    % G3 Dynamic Prompts: "When prompts are generated dynamically..., the process \must be thoroughly documented".
    \item \iconM Publish all prompts or, when using templates, prompt templates with representative instances, including their structure, content, formatting, and dynamic components, as \supplementarymaterial~(G3).
    % G3 Prompt Reporting / Supplementary: "researchers \must report all prompts that were used"; "When prompt templates are used, researchers \must report them alongside representative prompt instances"; "researchers \must make all prompts and prompt variations publicly available as part of their \supplementarymaterial".
    \item \iconS For ensemble architectures, explain the coordination logic between models in the \paper~(G3).
    % G3 Ensembles: "researchers \should describe the architecture that connects the models" and the coordination logic; "the \paper \must at least contain a high-level description, and details can be reported in the \supplementarymaterial". Coordination-logic specifics are S.
    \item \iconS Justify design decisions~(G3).
    % G3 Recommendations: "Researchers \should explain design decisions, particularly how the models were hosted and accessed... and which retrieval mechanisms were implemented".
    \item \iconS Where legally possible, release the source code of the implementation under an open-source license~(G3).
    % G3 Recommendations: "Where legally possible..., researchers \should release the source code of their implementation under an open-source license".
    \item \iconS Report data preprocessing, versioning, and update frequency for stored data used for context augmentation~(G3).
    % G3 Retrieval/Augmentation: "Stored data used for context augmentation \should be reported, including details on data preprocessing, versioning, and update frequency".
    \item \iconS Describe prompt development rationale and selection process~(G3).
    % G3 Prompt Development: "Researchers \should explain in the \paper how they developed the prompts and why they decided to follow certain prompting strategies"; "If multiple versions of a prompt were tested, researchers \should describe how these variations were evaluated".
    \item \iconS Document input handling and token optimization strategies when prompts are long or complex~(G3).
    % G3 Prompting Strategy: "researchers \should describe the strategies they used to handle input length constraints"; "Token optimization measures... \should also be documented".
    \item \iconS For user-authored prompts, describe how they were collected and analyzed~(G3).
    % G3 Dynamic/User-Authored Prompts: "For studies involving human participants who create or modify prompts, researchers \should describe how these prompts were collected and analyzed".
    \item \iconS If full prompt disclosure is not feasible, provide summaries or examples~(G3).
    % G3 Supplementary Material: "If the complete set of prompts cannot be included in the \paper, researchers \should provide summaries and representative examples".
    \item \iconS Report prompt revisions and pilot testing insights~(G3).
    % G3 Prompt Development: "Researchers \should report any instances in which LLMs were used to suggest prompt refinements"; "If such testing was conducted, researchers \should summarize key insights".
    \item \iconS Include context-file contents as \supplementarymaterial~(G3).
    % G3 Context Files: researchers "\should publish the full contents of those files as \supplementarymaterial".
    \item \iconS Include the complete list of tools and skills (names with one-line purposes), tool schemas, skill definitions, sub-agent definitions, and connected MCP servers as \supplementarymaterial~(G3).
    % G3 Tool Catalog: "Researchers \should include the complete list..., tool schemas, skill definitions, sub-agent definitions, and the identities of any connected Model Context Protocol (MCP) servers as \supplementarymaterial".
\end{itemize}

\noindent\textit{Session Traces}
\begin{itemize}[label=$\square$]
    \item \iconS Include full interaction logs (prompts and responses) as \supplementarymaterial if privacy and confidentiality can be ensured~(G4).
    % G4 Interaction Logs: "Researchers \should report the full interaction logs... as part of their \supplementarymaterial". (For S5 specifically the matrix elevates this to \must.)
    \item \iconS For agentic systems, include interaction logs covering exchanges between humans and the agent and between external tools and the agent (human-in-the-loop feedback, approvals, refinements) as \supplementarymaterial~(G4).
    % G4 Interaction Logs: "For agentic systems, interaction logs generalize to cover exchanges between humans and the agent as well as between external tools and the agent... These \should also be reported as \supplementarymaterial".
    \item \iconS For agentic systems, report tool-call traces (tool name, arguments, result, ordering) as \supplementarymaterial~(G4).
    % G4 Tool-Call Traces: "Researchers \should report the complete tool-call trace as \supplementarymaterial, including for each call the tool name, arguments, result, and its ordering".
    \item \iconS Report usage traces showing which skills, context files, sub-agents, and tools were actually picked up during a run as \supplementarymaterial~(G4).
    % G4 Usage Traces: "Researchers \should report usage traces as \supplementarymaterial showing which skills were invoked, which context files were read or injected, which sub-agents were delegated to, and which tools were called during each run".
    \item \iconS For agentic systems, report developed plans as \supplementarymaterial if available~(G4).
    % G4 Agentic Plans: "the developed plans \should be reported as \supplementarymaterial if available".
\end{itemize}

\noindent\textit{Human Validation}
\begin{itemize}[label=$\square$]
    \item \iconM If using human validation, define the measured construct (e.g., usability, maintainability) and describe the measurement instrument in the \paper~(G5).
    % G5 Study Design Considerations: "Authors \must clearly define in the \paper the constructs that the human and LLM annotators evaluate".
    \item \iconM When developing or adapting measurement instruments, share them~(G5).
    % G5 Study Design Considerations: "When designing custom instruments to assess LLM output (e.g., questionnaires, scales), researchers \must share their instruments".
    \item \iconS Consider human validation early in the study design and build on established reference models for human-LLM comparison~(G5).
    % G5 Recommendations: "researchers \should consider human validation early in the study design, not as an afterthought"; "Researchers \should use established reference models to compare humans with LLMs".
    \item \iconS Validate LLM judgments against human judgment, report aggregation methods, and assess human-LLM agreement~(G5).
    % G5 Subjective Judgment and Agreement: "the LLM output \should be evaluated against judgments aggregated from multiple human judges, and researchers \should clearly describe their aggregation method"; "Researchers \should report measures of IRA or IRR".
    \item \iconS Discuss and, where feasible, control for confounding factors~(G5).
    % G5 Subjective Judgment and Agreement: "Confounding factors \should be discussed and, where feasible, controlled for (e.g., by categorizing participants according to their level of experience or expertise)". Power analysis is intentionally excluded here: G5 source uses plain "may" ("Where applicable, researchers may perform a power analysis...") and the next sentence explicitly notes "limited established guidance specific to determining sample sizes for LLM-human comparisons" - so it does not warrant a \should-level checklist item.
\end{itemize}

\noindent\textit{Benchmarks and Metrics}
\begin{itemize}[label=$\square$]
    \item \iconM Justify all benchmark and metric choices in the \paper~(G7).
    % G7 Recommendations: researchers "\must briefly justify (in the \paper) why selected benchmarks and metrics are suitable for the given task or study".
    \item \iconM Explain in the \paper why the selected metrics are suitable for the specific study~(G7).
    % G7 Recommendations: "researchers \must motivate why they chose a certain metric or variant thereof for their particular study"; "Prior adoption alone does not constitute sufficient justification".
    \item \iconS Include an open LLM as a baseline when using commercial models and report inter-model agreement~(G6).
    % G6 Recommendations: "Empirical studies using LLMs in SE, especially those that target commercial tools or models, \should incorporate an open LLM as a baseline and report established metrics for inter-model agreement".
    \item \iconS Summarize benchmark structure, task types, and limitations~(G7).
    % G7 Recommendations: researchers "\should summarize the structure and tasks of the selected benchmark(s)" and "\should discuss the limitations of the selected benchmark(s)".
    \item \iconS Justify the number of experiment repetitions, for example through a power analysis or by monitoring convergence of descriptive statistics~(G7).
    % G7 Recommendations: "researchers \should justify their chosen number of repetitions, for example, through a power analysis... or by monitoring the convergence of descriptive statistics".
\end{itemize}

\noindent\textit{Reproducibility, Ethics, and Resources}
\begin{itemize}[label=$\square$]
    \item \iconM For studies involving sensitive data, discuss data governance mechanisms compliant with applicable jurisdictional obligations~(G8).
    % G8 Ethical \& Regulatory: "Studies involving sensitive data \must discuss data governance mechanisms tailored towards LLM environments, compliant with applicable juristical obligations".
    \item \iconS Justify LLM usage in light of its resource demands~(G8).
    % G8 Environmental \& Sustainability: "The study findings \should be discussed in the context of the resource-intensiveness of LLMs to justify the increased resource consumption over traditional approaches".
    \item \iconS Where full sharing of prompts, traces, or datasets is not feasible, share representative examples for partial replicability~(G8).
    % G8 Mitigation Strategies bullet: "Replication Packages covering prompt and architecture specifications, model outputs, and representative examples for partial replicability". The source bullet has no RFC 2119 marker; \iconS is interpretive, treating G8's mitigation list as \should-level given the framing "The following mitigation strategies can help address the threats". Concrete instances of "if full sharing not feasible, share representative examples" appear with explicit \should in G2 (fine-tuning data), G3 (prompts), and G4 (interaction logs); this item is the cross-artifact summary.
    \item \iconS Provide a full replication package with step-by-step instructions as \supplementarymaterial~(G6).
    % G6 Recommendations: "Researchers \should provide a complete replication package as part of their \supplementarymaterial, including clear step-by-step instructions on how to verify and reproduce the results".
\end{itemize}

\paragraph{\textbf{Results}}
\begin{itemize}[label=$\square$]
    \item \iconM If comparing models or tools, use appropriate inferential statistics (e.g., hypothesis tests, effect sizes) rather than relying solely on summary statistics~(G7).
    % G7 Recommendations: "If comparing models or tools, researchers \must use appropriate inferential statistics... rather than relying solely on differences in means or other summary statistics".
    \item \iconS Repeat experiments due to the inherent non-determinism of LLMs and report the result distribution using descriptive statistics~(G7).
    % G7 Recommendations: "Due to LLM non-determinism, researchers \should repeat experiments and report descriptive statistics of model or tool performance".
    \item \iconS Use traditional (non-LLM) baselines for comparison where possible~(G7).
    % G7 Recommendations: "researchers \should check whether a less resource-intensive approach... can serve as a baseline. If so, the LLM or LLM-based tool should be compared with such baselines using suitable metrics".
    \item \iconS Report established metrics to make study results comparable; additional metrics may be reported where appropriate~(G7).
    % G7 Recommendations: "researchers \should report established metrics whenever possible, as this allows secondary research. They can, of course, report additional metrics".
\end{itemize}

\paragraph{\textbf{Limitations and Threats to Validity}}
\begin{itemize}[label=$\square$]
    \item \iconM Describe measurement constructs and methods; disclose any data leakage risks and avoid leaking evaluation data into LLM improvement pipelines~(G8).
    % G8 Internal Validity / Construct Validity: "researchers \must discuss potential data leakage effects and their impact on results"; "researchers \must discuss how they ensured quality of subjective results".
    \item \iconM Acknowledge non-disclosed confidential or proprietary components as reproducibility limitations~(G3).
    % G3 Recommendations: "Researchers \mustnot omit critical architectural details that could affect reproducibility"; G3 Challenges: "For commercial tools, researchers \must report all available information and acknowledge unknown aspects as limitations".
    \item \iconM Transparently report study limitations, including the impact of non-determinism and generalizability constraints~(G8).
    % G8 Recommendations / Reliability: "Researchers \must make every effort to present the limitations of their work clearly without defensiveness or obfuscation"; "Researchers \must discuss the measures taken to increase reliability and reproducibility".
    \item \iconM Specify whether generalization across LLMs or across time was assessed, and discuss model and version differences~(G8).
    % G8 External Validity: "The limitations and mitigations used for external validity \must be discussed"; covers cross-model transfer and cross-time instability.
    \item \iconS Employ and report strategies to mitigate identified validity and reproducibility threats, such as replication packages, human validation, longitudinal re-runs, triangulation, and sensitivity analysis~(G8).
    % G8 Mitigation Strategies: lists Replication Packages, Human Validation, Longitudinal Re-Runs, Triangulation, Sensitivity Analysis as strategies (no explicit \must); "Researchers \should consider triangulation, reflexivity, audit trails, and peer debriefing as complementary measures".
\end{itemize}
